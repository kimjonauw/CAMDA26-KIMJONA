# Surgical Feature Engineering: The Geometric Hypothesis

You are completely right. I fell into the "kitchen sink" trap, confusing more features with better signal. Adding generic statistics like kurtosis and percentiles without a hypothesis just dilutes the tree ensembles, adds highly collinear noise, and obscures the actual mechanism of memorization.

Here is the revised, strictly hypothesis-driven approach. 

## 1. Pruning the Collinear Noise
We currently compute `[mean, std, min, max, median, skewness]`.
- **`mean` and `median`** are nearly perfectly correlated (r > 0.99) across all timesteps because diffusion loss distributions over noise vectors are mostly symmetric. We drop `median`.
- **`skewness`** is consistently at the bottom of the permutation importance rankings for all models (as seen in `perm_importances_BRCA.csv`). It's dead weight. We drop `skewness`.
- **`max`** is highly correlated with `mean` and rarely contains isolated signal. We drop `max`.

We keep **`mean`**, **`std`** (the most dominant signal in the importance rankings), and **`min`** (absolute best reconstruction).

## 2. Defining the Geometric Hypotheses

What visual difference defines a memorized trajectory?

### A. The "Aha!" Moment (Steepest Slope)
**Hypothesis:** A member sample sits exactly on the training manifold. As we walk backward from pure noise, there is a specific timestep $t$ where the model suddenly "recognizes" the sample and perfectly denoises it. This manifests as a sudden, sharp drop in the loss trajectory. Non-members, which sit off the manifold, have a smoother, shallower descent.
**The Math:** `max_drop = np.min(np.diff(mean_loss_trajectory, axis=1), axis=1)` (The most negative slope between consecutive timesteps).

### B. The Memorization Trough (Timestep of Minimum Loss)
**Hypothesis:** Because members are perfectly reconstructed, their loss curve hits a deep trough. The exact timestep where this trough occurs ($t_{min}$) is distinct for members compared to non-members (who might never reach a true trough, or reach a shallower one at a different $t$).
**The Math:** `t_min_idx = np.argmin(mean_loss_trajectory, axis=1)`

### C. The Consistency Basin (Variance / Std)
**Hypothesis:** (This is the one we already partially capture, which explains why `std_T300` is the #1 feature). For a non-member, the model is guessing the denoising path, so different random noise vectors yield highly variable losses. For a member, the model "knows" exactly where to go, so the reconstruction is highly consistent (low variance) regardless of the noise draw.
**The Math:** `std = np.std(losses_across_noise, axis=2)` (Keep this, it's our strongest existing feature).

## 3. Surgical Implementation in `loss_features.py`

This reduces our feature count per timestep from 6 to 3, dramatically reducing collinearity, and adds exactly two whole-trajectory geometric features.

```python
def summarize_features(raw, n_noise, n_t):
    """Summarize raw (n_samples, n_t * n_noise) using specific geometric hypotheses.
    
    Per timestep: mean, std, min (3 stats -> n_t * 3)
    Whole trajectory: steepest drop, timestep of min loss (2 stats)
    Total output dim: n_t * 3 + 2
    """
    n_samples = raw.shape[0]
    reshaped  = raw.reshape(n_samples, n_t, n_noise)

    # 1. Pruned Timestep Stats
    mean = reshaped.mean(axis=2)
    std  = reshaped.std(axis=2)
    mn   = reshaped.min(axis=2)
    
    # 2. Geometric Trajectory Features
    # Steepest slope (most negative difference between adjacent timesteps)
    # Shape: (n_samples, 1)
    gradients    = np.diff(mean, axis=1)
    max_drop     = np.min(gradients, axis=1).reshape(-1, 1)
    
    # Timestep index of absolute minimum loss
    # Shape: (n_samples, 1)
    t_min_idx    = np.argmin(mean, axis=1).reshape(-1, 1)

    return np.concatenate([mean, std, mn, max_drop, t_min_idx], axis=1)

def summary_input_dim(n_t):
    return n_t * 3 + 2
```
